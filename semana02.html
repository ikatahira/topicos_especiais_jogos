🧠 Teoria da Autodeterminação (SDT) Ryan & Deci, 2000

A SDT é a principal teoria que explica por que as pessoas jogam. Ela diz que todo ser humano tem 3 necessidades psicológicas básicas. Quando um jogo satisfaz essas necessidades, os jogadores ficam engajados de forma genuína — não apenas por recompensas externas.

⚔️

Competência

Sentir que está ficando melhor no jogo. Maestria, progresso visível, superar desafios.

Elden Ring: mortes = aprendizado
🗺️

Autonomia

Ter escolhas reais que importam. Sentir que as ações são suas, não impostas.

BotW: "vá para onde quiser"
🤝

Relacionamento

Conexão com outros jogadores ou personagens. Pertencimento a uma comunidade.

Among Us: confiança e traição
COMPETÊNCIA

Como implementar em jogos?

Curva de dificuldade bem calibrada, feedback claro sobre progresso, sistema de achievements, tutoriais adaptivos, dificuldade dinâmica (DDA).

// EXEMPLO UNITY Adaptar dificuldade dos inimigos via PlayerStats.winRate para manter o jogador no Flow
AUTONOMIA

Mais que só escolhas

Autonomia não é só "abrir mundo". Um jogo linear ainda satisfaz autonomia se o jogador sente que as ações são voluntárias e fazem sentido para ele.

// EXEMPLO Hollow Knight: linear mas cada rota parece descoberta sua
RELACIONAMENTO

Além do multiplayer

Relacionamento não precisa de outros jogadores. NPCs com personalidade, lore profundo, e comunidades ao redor do jogo também atendem essa necessidade.

// EXEMPLO Stardew Valley: NPCs com rotinas e memória criam vínculo real
🌊 Flow State — O Estado de "No Game" Csikszentmihalyi

Flow é quando você "perde a noção do tempo" jogando. É o estado ideal de engajamento — desafio e habilidade em perfeito equilíbrio. Todo bom game designer busca manter o jogador nessa zona.

⚡ Motivação Intrínseca vs Extrínseca

O perigo dos sistemas de recompensa mal projetados: quando você adiciona recompensas externas demais, pode destruir a motivação natural do jogador. Isso se chama Overjustification Effect.

🟢 Intrínseca

Joga pelo prazer da atividade em si

Motivação sustentada a longo prazo

Continua mesmo sem recompensas

Associada à satisfação das 3 necessidades SDT

🎮 Minecraft: constói porque é criativo, não pela recompensa

🟡 Extrínseca

Joga por recompensa externa (XP, moeda, ranking)

Eficaz no curto prazo, esgota com o tempo

Pode destruir motivação intrínseca existente

Base de muitos jogos mobile F2P

📱 Battle pass: joga obrigado para não perder o passe

⚠️ Regra de Ouro para Game Designers: Use recompensas externas para INICIAR o engajamento, mas construa sistemas que desenvolvam motivação intrínseca. Jogos que só têm FOMO (Fear Of Missing Out) como engajamento têm churn altíssimo.

Análise SDT de Jogo AAA
INDIVIDUAL RELATÓRIO
+100 XP
›

Escolha um jogo que você já jogou (BotW, Elden Ring, Celeste, Hollow Knight, qualquer AAA ou indie relevante). Analise profundamente como ele atende — ou falha em atender — às 3 necessidades da SDT: Competência, Autonomia e Relacionamento.

📦 ENTREGA

Relatório de 2-3 páginas com: (1) breve intro do jogo escolhido, (2) análise de cada dimensão SDT com exemplos específicos de mecânicas, (3) nota de 0-10 para cada dimensão com justificativa, (4) proposta de melhoria de 1 elemento com base na teoria.

💡 DICAS
  • Seja específico — não "o jogo tem progresso", mas "o sistema de runes em Elden Ring representa competência porque..."
  • Inclua screenshots das mecânicas que está analisando
  • Compare com a semana anterior: esse jogo é gamificado? Ou é só um jogo?

Implementação: Difficulty Adaptive System
INDIVIDUAL UNITY / UNREAL
+200 XP
›

Implemente um sistema que adapta a dificuldade em tempo real para manter o jogador na Flow Zone. Baseie-se em pelo menos 2 métricas de desempenho (ex: winRate, deathCount, tempoMédioNaFase). O sistema deve ser visível na UI (algum indicador do estado atual).

📦 ENTREGA

Repositório GitHub com: (1) protótipo jogável em Unity ou Unreal, (2) código comentado do DDA System, (3) README explicando a lógica do algoritmo e as métricas usadas, (4) GIF ou vídeo curto demonstrando o sistema funcionando.

💡 DICAS
  • Comece simples: winRate = vitorias / tentativas. Se > 0.7, aumenta dificuldade. Se < 0.3, diminui
  • Mostre na UI o "estado motivacional" atual: BORED / FLOW / ANXIOUS
  • Extra: use PlayerPrefs para salvar histórico e comparar sessões
03
Experimento: Intrínseca vs Extrínseca
GRUPO (3-4 pessoas) DOCUMENTO
+100 XP
›

Desenhe um experimento controlado que teste se adicionar recompensas externas aumenta ou diminui a motivação intrínseca de jogar algo específico. O experimento não precisa ser executado (apenas planejado com rigor), mas deve ser realista e baseado na teoria.

📦 ENTREGA

Documento de 2 páginas com: (1) hipótese clara, (2) metodologia — grupos controle e experimental, (3) métricas quantitativas escolhidas, (4) possíveis resultados esperados com base na teoria, (5) como esse experimento se aplicaria a um jogo real em fase de desenvolvimento.

💡 DICAS
  • Exemplo: Grupo A joga Tetris sem recompensas. Grupo B joga com sistema de pontos e ranking. Mede-se: sessões por semana, tempo por sessão, satisfação relatada
  • Inclua análise do Overjustification Effect — como você evitaria esse problema no design?
  • Conecte com games reais: qual jogo famoso teve esse problema? (Warcraft 3: Reforged, Diablo 4 no lançamento, etc)
