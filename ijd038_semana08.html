Atividade 1 — Análise Octalysis de App Real
+65 XP

Escolha um app gamificado (Duolingo, Strava, Nike Run Club, Habitica) e faça uma análise completa usando o framework Octalysis — pontuando cada Core Drive e identificando lacunas.

Passos
  1. Escolha o app e crie conta se não tiver — use por 30 minutos observando cada elemento gamificado
  2. Para cada um dos 8 Core Drives: dê uma nota de 0-10 e cite o elemento específico que ativa (ou a ausência)
  3. Crie um gráfico radar/spider mostrando o perfil Octalysis do app
  4. Identifique: qual Drive é mais forte? Qual está ausente? O que está faltando?
  5. Proponha 3 mudanças para ativar drives sub-utilizados com exemplos específicos
Entrega

Relatório 4 páginas com gráfico radar + análise de cada CD + 3 propostas de melhoria com mockups

👤
Atividade 2 — Personas e User Journeys por Player Type
+60 XP

Crie 4 personas detalhadas (uma por Player Type de Bartle) e mapeie a User Journey de cada uma em um sistema gamificado de sua escolha.

Passos
  1. Para cada Player Type (Achiever, Explorer, Socializer, Killer): crie uma persona com nome, foto, contexto, motivações e frustrações
  2. Escolha um sistema gamificado (pode ser app real ou hipotético) onde todas as 4 personas possam existir
  3. Para cada persona: mapeie a jornada em 6-8 passos desde descoberta até abandono ou retenção longa
  4. Identifique em cada passo: qual é a emoção? Qual Core Drive está ativo? Existe fricção?
  5. Conclua: o sistema serve bem a todos os Player Types? O que mudar para melhorar a jornada do tipo mais negligenciado?
Entrega

Documento com 4 personas detalhadas + 4 journey maps visuais (Miro, FigJam ou desenhado) + análise comparativa

🎮
Atividade 3 — Protótipo + Playtesting de Sistema Gamificado
+75 XP

Em grupo, projete, prototipe e playtest um sistema gamificado do zero — usando Octalysis e Player Types como guias. Itere baseado nos resultados do playtesting.

Passos
  1. Escolha o domínio: gamificar um comportamento real (leitura, exercício, reciclagem, socialização)
  2. Use Octalysis para mapear quais 4+ Core Drives o sistema deve ativar
  3. Prototipe em papel ou Figma: telas principais do sistema com os elementos gamificados
  4. Realize playtesting com 4 pessoas diferentes (tente incluir diferentes Player Types)
  5. Documente comportamentos não esperados e itere: o que mudou após o playtesting?
Entrega

Protótipo v1 + relatório de playtesting (o que foi observado) + protótipo v2 com mudanças + análise de Octalysis antes e depois
